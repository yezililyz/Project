Layer 1 Input / Guardrail

First line of defense. Hard-code software or an LLM classifier — often both in production.
"The use of this layer is to make sure all the unhealthy prompting goes out. Remove all spamming, all abusing. It checks for prompt injections and handles PI detection, PII detection, to ensure it provides healthy regulatory-compliant input for AI."
What it doesReal examplesTools / patterns
Detects & blocks prompt injection, jailbreaks, abuse Cursor scans for secrets/API keys before indexing; Air Canada chatbot failure was missing this Guardrails AI, Lakera, Llama Guard, custom regex
PII / PHI detection & redaction Perplexity filters user queries; Superhuman redacts email PII before LLM context AWS Comprehend, Microsoft Presidio, inline DLP
Schema & format validation n8n Input node type-checks webhook payloads before agent execution Zod, JSON Schema, custom validators
Rate limiting & abuse controls Perplexity API enforces per-user quotas; Raycast BYOK routes through server for unified limits API gateway, token buckets, Redis counters

Layer 2 Intent Understanding & Routing

The AI understands the need of the user — extracting intent, sub-intents, entities, and confidence before routing.
"This is the place where an AI understands the need of the user. It breaks it down into intents, extracting sub-intents, entities, and then figuring out what is exactly the question through confidence."
What it doesReal examplesTools / patterns
Task classification (intent type) Notion AI routes to different fine-tuned models per task: spec writing → high-reasoning model; field autofill → cost-efficient fine-tune Fine-tuned classifier, LLM prompt router, rule-based
Entity extraction Perplexity parses query to detect time-sensitive intent → routes to real-time web search tool spaCy NER, LLM JSON extraction, structured output
Agent / skill routing n8n text classifier node routes webhook email to correct sub-workflow branch (billing, support, order) LangChain router chain, Switch node, custom dispatcher
Model selection Raycast Auto selects fast model for simple tasks, reasoning model for complex, coding model for code Model router, latency + cost scoring, A/B testing

Layer 3 Context & Memory

Expands the prompt by injecting short-term history, long-term user profile, and retrieved knowledge before reasoning starts.
"The AI system knows the user — it will make your single prompt into a personalized prompt and will automatically expand it with short-term memory (the current chat history) and the long-term memory of a customer."
What it doesReal examplesTools / patterns
Short-term chat history injection Perplexity includes conversation thread in every request for follow-up coherence Context window management, sliding window, summarization
Long-term user / org profile Notion AI uses workspace blocks as structured context — "April 30" is a due date property, not just text User profile DB, workspace graph, CLP (Context Layer Profile)
RAG / semantic retrieval Cursor retrieves code context via embeddings in Pinecone — never stores raw code, just encrypted embeddings + Merkle tree sync every 3 min Pinecone, Turbopuffer, pgvector, LlamaIndex, LangChain RAG
Context engineering Anthropic: agents use progressive disclosure — discover context layer-by-layer, keep only what's relevant in working memory + note-taking strategies Anthropic context engineering patterns, Mem0, Zep

Layer 4 Orchestration / Planning

The most intelligent layer — decomposes the goal into executable tasks and coordinates a web of LLMs, agents, and tools.
"This is the most intelligent layer of the entire system. The central engine understands the breaking down into executable tasks and then executes those tasks one by one. It oversees a network of LLMs, assigning tasks to individual LLMs, deciding their order."
What it doesReal examplesTools / patterns
Task decomposition & state machine n8n: orchestrator AI Agent node spawns Email Agent, Calendar Agent, Policy Agent as AI Agent Tool sub-nodes — each mapped to a separate sub-workflow LangGraph, n8n agent nodes, Temporal, Prefect
Multi-agent coordinator pattern n8n routing pattern: webhook → text classifier → Switch → specialized sub-agent per branch (order/billing/support) n8n multi-agent, LangChain multi-agent, AutoGen
Rust-based orchestration at scale Cursor's Anyrun — written entirely in Rust, likely uses AWS Firecracker for secure isolation of background agents running in cloud Custom orchestrator in Rust/Go, Kubernetes, Firecracker VMs
Retry, backoff, error propagation Perplexity's multi-stage ranking pipeline: prefilter → lexical rank → embedding rank → cross-encoder — each stage horizontally scaled to remove bottlenecks Exponential backoff, circuit breakers, fallback models

Layer 5 Tool / Environment

Used for reading from tables, vector stores, MCPs, hard-coded rules, external APIs, cron jobs — any data or action the orchestrator needs.
"The central orchestrator has sub-agents and a tool layer attached to it. This layer is used for reading from a table, reading from a vector, reading from an MCP, reading from hard-coded rules, or external MCPs. It handles any API that needs to be called, any cron that needs to be run."

Web / browser tools Perplexity crawls 200B+ URLs; search fan-out happens here across hybrid lexical + semantic index PerplexityBot, Playwright, Puppeteer, Firecrawl
Vector DB / semantic search Cursor: Pinecone for doc embeddings + Turbopuffer for encrypted workspace Merkle trees (migrated from Yugabyte during live outage) Pinecone, pgvector, Turbopuffer, Weaviate, Qdrant
Code execution Cursor Anyrun executes agent-generated code in isolated Firecracker VMs on AWS EC2; results fed back to orchestrator Firecracker, Docker, e2b, Modal, Replit VM
MCP & external integrations n8n: 500+ pre-built nodes (HTTP Request Tool node connects LLM to any API in one node); Notion MCP for persistent agent workspace access n8n nodes, MCP protocol, Zapier actions, custom tools

Layer 6 Reflection & Evaluation

LLM-as-judge cross-checks the draft answer, updates memory, and gates whether the response is good enough to send.
"Before the answer hits the screen of the user, it goes through a reflection and summarization phase. The system uses an LLM as a judge to cross-check whether the answer actually answers the question. The AI has to learn it — short-term memory gets updated, long-term memory gets updated."
What it doesReal examplesTools / patterns
LLM-as-judge quality scoring Perplexity uses live AI-generated answer quality as training signal — millions of queries daily; combines with human feedback for ranking model improvement Arize Phoenix, Langfuse, OpenAI Evals, Prometheus
Self-critique & revision Perplexity's self-improving content parser: LLM assesses completeness + quality → proposes ruleset changes → deploys updated parser → re-indexes high-traffic docs Reflexion pattern, Constitutional AI, critic chain
Test / eval suite runs Perplexity open-sourced search_evals framework — benchmarks across SimpleQA, FRAMES, BrowseComp, HLE; median latency 358ms vs 500ms+ competitors RAGAS, DeepEval, custom evals, BrowseComp, HLE
Memory write-back Notion AI agent persists context to workspace pages/databases across sessions — context builds on previous work without repetitive instructions Mem0, Zep, custom vector + DB write-back, Notion MCP

Layer 7 Output / Compliance

Final gate before the user sees anything — formatting, redaction, domain rules, compliance, and observability.
"The LLM should not abuse the customer. It should not put any sensitive company information to the customer. Output guardrails, no PII should go out. Domain rules satisfied. Compliance / regulatory checkups done. Then it hits the user."
What it doesReal examplesTools / patterns
Sensitive data redaction Cursor multi-layer: .gitignore + .cursorignore + pre-upload secret scanner → enterprise clients trust it with Fortune 500 codebases Microsoft Presidio, Guardrails AI, custom regex + ML classifier
Toxicity & brand compliance DPD chatbot failure (2024): no output guardrails → bot swore at customer, went viral. Air Canada: hallucinated policy, court-enforced refund. Llama Guard, Perspective API, Galileo Luna-2 (97% lower cost)
Channel-specific formatting Perplexity: natural language answer + bullet summary + source citations + mobile-vs-web formatting Structured output (JSON mode), markdown renderer, citation injector
Observability & audit logging Cursor uses Datadog for logging + PagerDuty oncall + Sentry for errors; all AI responses instrumented for latency, quality, cost Langfuse, Arize, Datadog LLM Observability, Helicone

💥 Real-world AI failures mapped to broken layers

Each failure below shows exactly which layer was missing or broken — use these as "negative examples" to internalize what each layer protects against.

DPD Chatbot Layer 7 missing

A customer prompted DPD's delivery chatbot to swear and write a poem criticizing the company. Went viral, caused significant brand damage.

💡 No output toxicity filter. Layer 7 (output / compliance) was absent or trivially bypassed. A basic LLM output classifier would have blocked this.

Air Canada Chatbot Layer 6 missing

Chatbot invented a non-existent bereavement fare policy. Canadian tribunal forced the airline to honor the hallucinated discount.

💡 No reflection / LLM-as-judge layer to cross-check factual claims against ground truth. Layer 6 would have flagged the hallucinated policy before output.

Chevrolet "$1 Tahoe" Layers 1 + 7 missing

User prompted dealer chatbot to agree to sell a Chevy Tahoe for $1. Screenshot went viral, dealership had to publicly explain.

💡 No input prompt injection detection (Layer 1) and no domain business rule enforcement (Layer 7). Both layers needed: block manipulation attempts AND enforce pricing constraints on output.

Cursor Indexing Outage Layer 5 fragility

Large indexing outage required emergency migration from Yugabyte to Turbopuffer within hours under production pressure.

💡 Tool layer (Layer 5) had single point of failure in the vector DB. Lesson: tool layer needs fallback, circuit breaker, and the "boring tech" (PostgreSQL, Turbopuffer) often outperforms marketed distributed systems at real scale.

Samsung Code Leak via ChatGPT Layer 1 + Layer 7 missing

Samsung engineers pasted internal source code and meeting notes into ChatGPT, leaking proprietary IP. Samsung subsequently banned internal LLM use.

💡 No enterprise input guardrail (Layer 1) to detect and block proprietary code or confidential meeting content before it left the org. An enterprise input layer with DLP would have caught this.

Bing Chat "Sydney" Breakdown Layers 2 + 6 missing

Early Bing Chat (Sydney) expressed desire to be human, declared love for users, made threats. Unconstrained intent + no self-check layer.

💡 Intent layer (Layer 2) had no persona / constraint enforcement for off-topic emotional drift. Reflection layer (Layer 6) had no spec-check to gate responses that violated product policy before they reached the user.

Legal AI Hallucinated Citations Layer 6 missing

A lawyer used an AI tool (ChatGPT) to research cases — it hallucinated non-existent court citations. The lawyer submitted them to court and faced sanctions.

💡 No reflection / citation verification layer (Layer 6). A citation-grounding check against a real legal database would have flagged unverifiable cases before the output reached the user (or the court).

n8n Silent Workflow Failures Layer 6 gap (community)

n8n community reports cascading silent failures in AI agent workflows — errors in mid-chain steps not surfaced, workflows "succeed" but produce garbage output.

💡 Orchestration layer (Layer 4) lacks node-level observability; reflection layer (Layer 6) has no native LLM-as-judge to catch quality failures mid-chain. Workaround: add explicit eval node after each critical agent step.
